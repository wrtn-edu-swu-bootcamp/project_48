# 대학생 대상 웹 서비스 디자인 가이드

## 개요
본 디자인 가이드는 대학생을 대상으로 하는 웹 서비스의 일관된 디자인 시스템을 정의합니다.
학교 공식 서비스처럼 신뢰감 있고, 심플하며, 학생 친화적인 디자인을 목표로 합니다.

**메인 컬러**: 버건디 (Burgundy) `#800020`  
**서브 컬러**: 연한 회색 계열  
**디자인 철학**: 심플하고 학교 공식 서비스 같은 느낌

---

## 1. 디자인 철학 및 원칙

### 1.1 디자인 철학
- **신뢰감**: 대학교 공식 서비스처럼 신뢰할 수 있는 느낌
- **명확성**: 정보가 명확하게 전달되는 구조
- **접근성**: 모든 사용자가 쉽게 사용할 수 있는 인터페이스
- **일관성**: 전체 서비스에서 일관된 디자인 언어 사용
- **미니멀리즘**: 불필요한 장식 요소 제거, 핵심에 집중

### 1.2 디자인 원칙

#### 가독성 최우선
- 충분한 글자 크기 (모바일 최소 16px)
- 적절한 여백 (요소 간 최소 16px)
- 명확한 계층 구조

#### 정보 과다 배치 금지
- 한 화면에 핵심 정보만 표시
- 스크롤을 통한 자연스러운 정보 탐색
- 중요도에 따른 시각적 강조

#### 명확한 시각적 구분
- 버튼과 입력창이 명확하게 구분됨
- 인터랙티브 요소와 정적 콘텐츠 구분
- 상태 변화가 명확하게 표시됨

#### 사용자 중심 설계
- 직관적인 네비게이션
- 예측 가능한 인터랙션
- 명확한 피드백 제공

## 2. 컬러 시스템

### 2.1 메인 컬러

#### 버건디 (Burgundy) `#800020`
**사용 용도:**
- 주요 버튼 배경
- 강조 요소 (링크, 아이콘)
- 브랜드 아이덴티티 표현
- 활성 상태 표시

**변형 색상:**
- **Hover 상태**: `#990025` (약간 밝게)
- **Active 상태**: `#660019` (약간 어둡게)
- **Disabled 상태**: `#CCCCCC` (회색)

### 2.2 서브 컬러 (연한 회색 계열)

#### 배경 컬러
- **주 배경**: `#FFFFFF` (화이트)
- **보조 배경**: `#FAFAFA` (매우 연한 회색)
- **구분선 배경**: `#F5F5F5` (연한 회색)
- **카드/컨테이너 배경**: `#FFFFFF` 또는 `#FAFAFA`

#### 텍스트 컬러
- **주요 텍스트**: `#1A1A1A` (거의 검정)
- **보조 텍스트**: `#666666` (중간 회색)
- **비활성 텍스트**: `#999999` (연한 회색)
- **플레이스홀더**: `#CCCCCC` (매우 연한 회색)

#### 구분선 컬러
- **주 구분선**: `#E5E5E5` (연한 회색)
- **보조 구분선**: `#F0F0F0` (매우 연한 회색)

### 2.3 상태 컬러

#### 성공 (Success)
- **메인**: `#10B981` (초록)
- **배경**: `#D1FAE5` (연한 초록)

#### 경고 (Warning)
- **메인**: `#F59E0B` (주황)
- **배경**: `#FEF3C7` (연한 주황)

#### 오류 (Error)
- **메인**: `#EF4444` (빨강)
- **배경**: `#FEE2E2` (연한 빨강)

#### 정보 (Info)
- **메인**: `#3B82F6` (파랑)
- **배경**: `#DBEAFE` (연한 파랑)

### 2.4 컬러 사용 가이드

#### 금지 사항
- 메인 컬러와 유사한 강렬한 색상 혼용 금지
- 과도한 그라데이션 사용 금지
- 배경에 강렬한 색상 사용 금지
- **색상만으로 정보 전달 금지** (아이콘, 라벨, 패턴 추가 필요)

#### 권장 사항
- 배경은 화이트 또는 연한 회색 계열 사용
- 강조는 버건디 컬러로 제한
- 상태 표시는 표준 상태 컬러 사용
- **시맨틱 컬러 토큰 사용 권장** (예: `color-primary`, `color-background-secondary`)

### 2.5 색상 대비 요구사항

#### WCAG 대비 기준
- **일반 텍스트**: 최소 4.5:1 대비 (WCAG AA)
- **큰 텍스트 또는 굵은 텍스트**: 최소 3:1 대비 (18px 이상 또는 Bold)
- **인터랙티브 요소**: 최소 3:1 대비 (버튼, 링크 등)

#### 대비 확인 방법
- 모든 텍스트와 배경 색상 조합을 대비 도구로 확인
- 자동화된 접근성 검사 도구 사용 (예: WebAIM Contrast Checker)
- 다양한 디스플레이에서 실제 테스트

### 2.6 컬러 스페이스 고려사항

#### 지각적으로 균일한 컬러 스페이스
- 향후 확장 시 OKLCH 같은 지각적으로 균일한 컬러 스페이스 고려
- 대비 요구사항 준수하면서 브랜드 색상 유지 가능
- 라이트/다크 모드 전환 시 색상 조정 용이

## 2.5 디자인 토큰 시스템 (Design Tokens)

### 2.5.1 토큰 계층 구조

디자인 토큰은 디자인 시스템의 기본 구성 요소를 체계적으로 관리하기 위한 방법입니다.

#### 1단계: Primitives (원시 토큰)
가장 기본적인 디자인 값들:
- **색상**: `#800020`, `#FFFFFF`, `#1A1A1A` 등
- **타이포그래피**: 폰트 크기, 웨이트, 행간
- **간격**: 4px, 8px, 16px 등
- **반경**: 8px, 12px 등
- **모션**: 트랜지션 시간, 이징 함수

#### 2단계: Semantic Aliases (시맨틱 별칭)
의미를 가진 토큰 이름:
- **색상**: `color-primary`, `color-text-on-primary`, `color-background-secondary`
- **타이포그래피**: `font-size-heading-1`, `font-weight-bold`
- **간격**: `spacing-md`, `spacing-section`

#### 3단계: Component-specific Tokens (컴포넌트별 토큰)
특정 컴포넌트에 사용되는 토큰:
- `button-primary-background`
- `input-border-color`
- `card-padding`

### 2.5.2 시맨틱 컬러 토큰 사용 권장

#### 권장 방식
```css
/* 좋은 예: 시맨틱 토큰 */
.button-primary {
  background-color: var(--color-primary);
  color: var(--color-text-on-primary);
}

/* 피해야 할 방식: 모호한 이름 */
.button-primary {
  background-color: var(--blue-100);
  color: var(--gray-50);
}
```

#### 토큰 명명 규칙
- 역할 기반: `color-primary`, `color-success`, `color-error`
- 상태 기반: `color-hover`, `color-active`, `color-disabled`
- 용도 기반: `color-background`, `color-text`, `color-border`

### 2.5.3 테마 시스템 준비

#### 라이트 모드 (기본)
- 현재 정의된 컬러 시스템 사용
- 화이트 배경, 다크 텍스트

#### 다크 모드 (향후 확장)
- 다크 배경, 라이트 텍스트
- 버건디 메인 컬러는 유지하되 밝기 조정
- 모든 대비 기준 충족

#### 고대비 모드 (향후 확장)
- 최대 대비를 위한 색상 조정
- WCAG AAA 기준 준수

#### Comfort Mode (커스터마이징 옵션)
- 사용자가 색상, 폰트 크기, 대비 등을 조정 가능
- 신경다양성 학생을 위한 개인화 옵션

## 3. 타이포그래피

### 3.1 폰트 패밀리

#### 한글 폰트
- **주 폰트**: 'Noto Sans KR', 'Malgun Gothic', sans-serif
- **대체 폰트**: 'Apple SD Gothic Neo', 'Nanum Gothic', sans-serif

#### 영문/숫자 폰트
- **주 폰트**: 'Inter', 'Roboto', sans-serif
- **대체 폰트**: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif

#### 코드/모노스페이스
- **폰트**: 'Fira Code', 'Consolas', 'Monaco', monospace

### 3.2 폰트 크기 시스템

#### 제목 (Headings)
- **H1**: 32px / 1.2 (Bold) - 메인 페이지 제목
- **H2**: 28px / 1.3 (Bold) - 섹션 제목
- **H3**: 24px / 1.3 (Bold) - 서브 섹션 제목
- **H4**: 20px / 1.4 (SemiBold) - 카드 제목
- **H5**: 18px / 1.4 (SemiBold) - 작은 제목
- **H6**: 16px / 1.5 (SemiBold) - 최소 제목

#### 본문 (Body)
- **Large**: 18px / 1.6 (Regular) - 강조 본문
- **Base**: 16px / 1.6 (Regular) - 기본 본문
- **Small**: 14px / 1.5 (Regular) - 보조 텍스트
- **XSmall**: 12px / 1.4 (Regular) - 캡션, 라벨

### 3.3 폰트 웨이트

- **Light**: 300 - 사용하지 않음
- **Regular**: 400 - 본문 기본
- **Medium**: 500 - 강조 본문
- **SemiBold**: 600 - 작은 제목
- **Bold**: 700 - 제목, 강조

### 3.4 타이포그래피 사용 가이드

#### 제목 사용
- H1은 페이지당 1개만 사용
- 제목 계층 구조 명확히 유지
- 제목과 본문 간 충분한 여백 (최소 24px)

#### 본문 사용
- 본문 텍스트는 16px 이상 유지 (가독성 권장: 16-18px)
- 긴 텍스트는 적절한 행간 (1.6) 유지
- 중요한 정보는 Bold 또는 색상으로 강조
- **단락 간격**: 최소 2배 폰트 크기 (예: 16px 폰트 → 32px 간격)

#### 텍스트 정렬
- 본문은 좌측 정렬 (Left Align)
- 제목은 좌측 정렬 또는 중앙 정렬
- 숫자, 코드는 우측 정렬 가능

### 3.5 타이포그래피 스케일 (Type Scale)

#### 비율 기반 스케일
일관된 타이포그래피 계층을 위한 비율:
- **Minor Third**: ≈1.2 (가장 작은 차이)
- **Major Third**: ≈1.25 (권장)
- **Perfect Fourth**: ≈1.333 (더 큰 차이)

#### 현재 시스템 비율
- H1 (32px) → H2 (28px): ≈1.14
- H2 (28px) → H3 (24px): ≈1.17
- H3 (24px) → H4 (20px): 1.2
- H4 (20px) → Body (16px): 1.25

### 3.6 단위 사용 가이드

#### 상대 단위 권장
- **rem**: 루트 폰트 크기 기준 (권장)
- **em**: 부모 요소 기준
- **px**: 고정 크기가 필요한 경우만 사용

#### 사용자 확대/축소 지원
- 상대 단위 사용으로 브라우저 확대 기능 지원
- 최소 200%까지 확대 가능하도록 설계
- 레이아웃이 깨지지 않도록 반응형 디자인 적용

### 3.7 텍스트 간격 조정 지원

#### 사용자 커스터마이징 지원 (WCAG 요구사항)
다음 간격 조정을 지원해야 합니다:
- **행간 (Line Height)**: 최소 1.5배까지 증가 가능
- **단락 간격**: 최소 2배 폰트 크기
- **글자 간격 (Letter Spacing)**: ~0.12 × font-size까지 증가 가능
- **단어 간격 (Word Spacing)**: 최소 ~0.16 × font-size

#### 구현 방법
```css
/* 기본 스타일 */
.text-content {
  line-height: 1.6;
  letter-spacing: normal;
  word-spacing: normal;
}

/* 사용자 조정 지원 */
.text-content {
  line-height: clamp(1.5, var(--user-line-height, 1.6), 2.5);
  letter-spacing: clamp(0, var(--user-letter-spacing, 0), 0.12em);
  word-spacing: clamp(0, var(--user-word-spacing, 0), 0.16em);
}
```

### 3.8 접근성 향상 폰트

#### 고가독성 폰트 옵션
- **Atkinson Hyperlegible**: 저시력 사용자를 위한 폰트
- 명확한 글자 구분, 열린 형태
- 향후 확장 시 고려 사항

#### 현재 폰트 시스템
- Noto Sans KR: 한글 가독성 우수
- Inter: 영문/숫자 가독성 우수
- 두 폰트 모두 명확한 글자 구분 제공

## 4. 간격 시스템 (Spacing)

### 4.1 간격 단위
기본 단위: **4px** (0.25rem)  
**8포인트 그리드 시스템**: 모든 간격은 8px의 배수로 확장 가능

### 4.2 간격 스케일

- **XS**: 4px (0.25rem) - 요소 내부 최소 간격
- **SM**: 8px (0.5rem) - 작은 요소 간 간격, 8포인트 그리드 기본 단위
- **MD**: 16px (1rem) - 기본 요소 간 간격
- **LG**: 24px (1.5rem) - 섹션 내 간격
- **XL**: 32px (2rem) - 섹션 간 간격
- **2XL**: 48px (3rem) - 큰 섹션 간격
- **3XL**: 64px (4rem) - 페이지 레벨 간격

### 4.3 간격 사용 가이드

#### 요소 내부 간격 (Padding)
- 작은 요소: 8px ~ 12px
- 중간 요소: 16px ~ 20px
- 큰 요소: 24px ~ 32px

#### 요소 간 간격 (Margin)
- 관련 요소: 8px ~ 16px
- 독립 요소: 24px ~ 32px
- 섹션 간: 48px ~ 64px

#### 인라인 vs 수직 스택 간격
- **인라인 간격 (Gap)**: 가로 배치 요소 간 간격, 8px ~ 16px
- **수직 스택 간격**: 세로 배치 요소 간 간격, 16px ~ 24px
- **텍스트 간격**: 단락 간 최소 2배 폰트 크기

#### 최소 인터랙티브 요소 간격
- 터치 타겟 간 최소 간격: 8px
- 버튼 간 간격: 8px ~ 16px
- 입력 필드 간 간격: 16px ~ 24px

#### 반응형 간격
- 모바일: 기본 간격
- 태블릿: 기본 간격 × 1.25
- 데스크톱: 기본 간격 × 1.5

### 4.4 수직 리듬 (Vertical Rhythm)

#### 일관된 간격 관계 유지
- 제목과 본문 간: 최소 24px
- 단락 간: 최소 32px (16px 폰트 기준)
- 섹션 간: 최소 48px

#### 텍스트 간격 조정 지원
사용자가 다음 간격을 조정할 수 있어야 합니다:
- 행간 (Line Height): 최소 1.5배
- 단락 간격: 최소 2배 폰트 크기
- 단어 간격: 최소 ~0.16 × font-size
- 글자 간격: ~0.12 × font-size까지 증가 가능

## 5. 레이아웃 시스템

### 5.1 그리드 시스템

#### 컨테이너 너비
- **모바일**: 100% (좌우 패딩 16px)
- **태블릿**: 최대 768px, 중앙 정렬
- **데스크톱**: 최대 1200px, 중앙 정렬
- **와이드**: 최대 1440px, 중앙 정렬

#### 컬럼 시스템
- **모바일**: 1컬럼
- **태블릿**: 2컬럼 (필요 시)
- **데스크톱**: 12컬럼 그리드
- **간격**: 컬럼 간 24px (모바일 16px)

## 9. 애니메이션 및 트랜지션

### 9.1 트랜지션 속도

#### 빠름 (Fast)
- **시간**: 150ms
- **사용**: 호버 효과, 색상 변화

#### 보통 (Normal)
- **시간**: 200ms ~ 300ms
- **사용**: 기본 트랜지션, 페이드 인/아웃

#### 느림 (Slow)
- **시간**: 400ms ~ 500ms
- **사용**: 페이지 전환, 모달 열기/닫기

### 9.2 이징 함수
- **기본**: `ease-in-out`
- **부드러움**: `cubic-bezier(0.4, 0, 0.2, 1)`
- **바운스**: 사용하지 않음

### 9.3 애니메이션 가이드

#### 허용되는 애니메이션
- 페이드 인/아웃 (0.2초)
- 슬라이드 인/아웃 (0.3초)
- 색상 변화 (0.15초)
- 로딩 스피너 회전
- 마이크로 인터랙션 (호버 효과, 부드러운 트랜지션)

#### 금지되는 애니메이션
- 과도한 바운스 효과
- 장식적인 애니메이션
- 자동 재생 비디오/이미지
- 깜빡이는 효과 (특정 주파수 이상, WCAG 위반)

### 9.4 모션 감소 옵션 (Reduced Motion)

#### prefers-reduced-motion 지원
사용자가 모션 감소를 선호하는 경우 애니메이션을 비활성화해야 합니다.

```css
/* 기본 애니메이션 */
.button {
  transition: background-color 0.15s ease-in-out;
}

/* 모션 감소 옵션 */
@media (prefers-reduced-motion: reduce) {
  .button {
    transition: none;
  }
}
```

#### 애니메이션 비활성화 옵션
- 사용자 설정에서 애니메이션 비활성화 가능
- 모든 트랜지션과 애니메이션 제거 또는 최소화
- 필수적인 피드백만 유지 (예: 포커스 표시)

### 9.5 마이크로 인터랙션 가이드

#### 허용되는 마이크로 인터랙션
- **호버 효과**: 버튼 색상 변화, 그림자 증가
- **포커스 표시**: 명확한 아웃라인
- **클릭 피드백**: 약간의 스케일 효과 (scale: 0.95)
- **로딩 상태**: 스피너, 타이핑 인디케이터

#### 원칙
- 비방해적: 사용자의 작업을 방해하지 않음
- 빠름: 150ms ~ 300ms 이내
- 의미 있음: 상태 변화를 명확히 전달
- 접근성: 모션 감소 옵션 존중

## 11. 접근성 (Accessibility)

### 11.1 색상 대비
- **WCAG AA 기준**: 최소 4.5:1 (일반 텍스트)
- **WCAG AAA 기준**: 최소 7:1 (작은 텍스트)
- 색상만으로 정보 전달 금지

### 11.2 키보드 네비게이션
- 모든 인터랙티브 요소에 키보드 접근 가능
- 포커스 표시 명확 (버건디 테두리 2px)
- Tab 순서 논리적
- Esc 키로 모달/메뉴 닫기

### 11.3 스크린 리더 지원
- 의미 있는 HTML 시맨틱 태그 사용
- ARIA 레이블 적절히 사용
- 이미지에 alt 텍스트 제공
- 폼 요소에 라벨 연결

### 11.4 텍스트 크기
- 사용자 브라우저 설정 존중
- 최소 폰트 크기 16px
- **확대/축소 기능 지원**: 최대 200%까지 확대 가능
- 레이아웃이 깨지지 않도록 반응형 디자인 적용

### 11.5 법적 요구사항 및 준수

#### DOJ 규칙 (2024년 4월)
- **공립 대학 웹사이트**: 2026년 4월 24일까지 WCAG 2.1 Level A 및 AA 준수 필수
- **적용 범위**: 웹사이트뿐만 아니라 PDF, Google Docs, 임베드 미디어, 써드파티 도구 포함
- **작은 기관**: 2027년 4월 26일까지 준수

#### WCAG 준수 수준
- **최소 기준**: WCAG 2.1 Level AA
- **준비 중**: WCAG 2.2 준비 및 향후 WCAG 3.0 대비
- **정기적 감사**: 연간 최소 1회 접근성 감사 실시

### 11.6 텍스트 간격 조정 지원

#### 필수 지원 사항 (WCAG 요구사항)
다음 텍스트 간격 조정을 지원해야 합니다:
- **행간 (Line Height)**: 최소 1.5배까지 증가 가능
- **단락 간격**: 최소 2배 폰트 크기
- **글자 간격 (Letter Spacing)**: ~0.12 × font-size까지 증가 가능
- **단어 간격 (Word Spacing)**: 최소 ~0.16 × font-size

#### 구현 방법
CSS 변수와 clamp 함수를 사용하여 사용자 설정 지원:
```css
.text-content {
  line-height: clamp(1.5, var(--user-line-height, 1.6), 2.5);
  letter-spacing: clamp(0, var(--user-letter-spacing, 0), 0.12em);
  word-spacing: clamp(0, var(--user-word-spacing, 0), 0.16em);
}
```

### 11.7 포커스 상태 강화

#### 명확한 포커스 표시
- **테두리**: 버건디 `#800020`, 2px
- **대비**: 최소 3:1 대비 유지
- **시각적 표시**: 아웃라인 또는 박스 섀도우
- **키보드 사용자**: 항상 포커스 표시 유지

#### Skip Links
- 주요 콘텐츠로 바로 이동하는 링크 제공
- 키보드 사용자의 네비게이션 효율성 향상
