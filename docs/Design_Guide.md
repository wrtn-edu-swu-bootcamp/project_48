# 대학생 대상 웹 서비스 디자인 가이드

## 개요
본 디자인 가이드는 대학생을 대상으로 하는 웹 서비스의 일관된 디자인 시스템을 정의합니다.
학교 공식 서비스처럼 신뢰감 있고, 심플하며, 학생 친화적인 디자인을 목표로 합니다.

**메인 컬러**: 버건디 (Burgundy) `#800020`  
**서브 컬러**: 연한 회색 계열  
**디자인 철학**: 심플하고 학교 공식 서비스 같은 느낌

---

## 1. 디자인 철학 및 원칙

### 1.1 디자인 철학
- **신뢰감**: 대학교 공식 서비스처럼 신뢰할 수 있는 느낌
- **명확성**: 정보가 명확하게 전달되는 구조
- **접근성**: 모든 사용자가 쉽게 사용할 수 있는 인터페이스
- **일관성**: 전체 서비스에서 일관된 디자인 언어 사용
- **미니멀리즘**: 불필요한 장식 요소 제거, 핵심에 집중

### 1.2 디자인 원칙

#### 가독성 최우선
- 충분한 글자 크기 (모바일 최소 16px)
- 적절한 여백 (요소 간 최소 16px)
- 명확한 계층 구조

#### 정보 과다 배치 금지
- 한 화면에 핵심 정보만 표시
- 스크롤을 통한 자연스러운 정보 탐색
- 중요도에 따른 시각적 강조

#### 명확한 시각적 구분
- 버튼과 입력창이 명확하게 구분됨
- 인터랙티브 요소와 정적 콘텐츠 구분
- 상태 변화가 명확하게 표시됨

#### 사용자 중심 설계
- 직관적인 네비게이션
- 예측 가능한 인터랙션
- 명확한 피드백 제공

## 3. 타이포그래피

### 3.1 폰트 패밀리

#### 한글 폰트
- **주 폰트**: 'Noto Sans KR', 'Malgun Gothic', sans-serif
- **대체 폰트**: 'Apple SD Gothic Neo', 'Nanum Gothic', sans-serif

#### 영문/숫자 폰트
- **주 폰트**: 'Inter', 'Roboto', sans-serif
- **대체 폰트**: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif

#### 코드/모노스페이스
- **폰트**: 'Fira Code', 'Consolas', 'Monaco', monospace

### 3.2 폰트 크기 시스템

#### 제목 (Headings)
- **H1**: 32px / 1.2 (Bold) - 메인 페이지 제목
- **H2**: 28px / 1.3 (Bold) - 섹션 제목
- **H3**: 24px / 1.3 (Bold) - 서브 섹션 제목
- **H4**: 20px / 1.4 (SemiBold) - 카드 제목
- **H5**: 18px / 1.4 (SemiBold) - 작은 제목
- **H6**: 16px / 1.5 (SemiBold) - 최소 제목

#### 본문 (Body)
- **Large**: 18px / 1.6 (Regular) - 강조 본문
- **Base**: 16px / 1.6 (Regular) - 기본 본문
- **Small**: 14px / 1.5 (Regular) - 보조 텍스트
- **XSmall**: 12px / 1.4 (Regular) - 캡션, 라벨

### 3.3 폰트 웨이트

- **Light**: 300 - 사용하지 않음
- **Regular**: 400 - 본문 기본
- **Medium**: 500 - 강조 본문
- **SemiBold**: 600 - 작은 제목
- **Bold**: 700 - 제목, 강조

### 3.4 타이포그래피 사용 가이드

#### 제목 사용
- H1은 페이지당 1개만 사용
- 제목 계층 구조 명확히 유지
- 제목과 본문 간 충분한 여백 (최소 24px)

#### 본문 사용
- 본문 텍스트는 16px 이상 유지 (가독성)
- 긴 텍스트는 적절한 행간 (1.6) 유지
- 중요한 정보는 Bold 또는 색상으로 강조

#### 텍스트 정렬
- 본문은 좌측 정렬 (Left Align)
- 제목은 좌측 정렬 또는 중앙 정렬
- 숫자, 코드는 우측 정렬 가능

## 4. 간격 시스템 (Spacing)

### 4.1 간격 단위
기본 단위: **4px** (0.25rem)

### 4.2 간격 스케일

- **XS**: 4px (0.25rem) - 요소 내부 최소 간격
- **SM**: 8px (0.5rem) - 작은 요소 간 간격
- **MD**: 16px (1rem) - 기본 요소 간 간격
- **LG**: 24px (1.5rem) - 섹션 내 간격
- **XL**: 32px (2rem) - 섹션 간 간격
- **2XL**: 48px (3rem) - 큰 섹션 간격
- **3XL**: 64px (4rem) - 페이지 레벨 간격

### 4.3 간격 사용 가이드

#### 요소 내부 간격 (Padding)
- 작은 요소: 8px ~ 12px
- 중간 요소: 16px ~ 20px
- 큰 요소: 24px ~ 32px

#### 요소 간 간격 (Margin)
- 관련 요소: 8px ~ 16px
- 독립 요소: 24px ~ 32px
- 섹션 간: 48px ~ 64px

#### 반응형 간격
- 모바일: 기본 간격
- 태블릿: 기본 간격 × 1.25
- 데스크톱: 기본 간격 × 1.5

## 5. 레이아웃 시스템

### 5.1 그리드 시스템

#### 컨테이너 너비
- **모바일**: 100% (좌우 패딩 16px)
- **태블릿**: 최대 768px, 중앙 정렬
- **데스크톱**: 최대 1200px, 중앙 정렬
- **와이드**: 최대 1440px, 중앙 정렬

#### 컬럼 시스템
- **모바일**: 1컬럼
- **태블릿**: 2컬럼 (필요 시)
- **데스크톱**: 12컬럼 그리드
- **간격**: 컬럼 간 24px (모바일 16px)

### 5.2 브레이크포인트

```css
/* 모바일 */
@media (min-width: 320px) { }

/* 태블릿 */
@media (min-width: 768px) { }

/* 데스크톱 */
@media (min-width: 1024px) { }

/* 와이드 */
@media (min-width: 1440px) { }
```

### 5.3 레이아웃 패턴

#### 기본 레이아웃
```
┌─────────────────────────────────┐
│         Header (고정)            │
├─────────────────────────────────┤
│      Main Content Area           │
│  (최대 너비: 1200px, 중앙 정렬) │
├─────────────────────────────────┤
│         Footer                   │
└─────────────────────────────────┘
```

#### 카드 레이아웃
- 카드 간 간격: 16px ~ 24px
- 카드 내부 패딩: 20px ~ 24px
- 카드 그림자: `0 2px 8px rgba(0, 0, 0, 0.08)`

- 사용: 작은 요소, 구분선 대체

#### SM (작은 그림자)
- `0 2px 4px rgba(0, 0, 0, 0.08)`
- 사용: 버튼, 작은 카드

#### MD (중간 그림자)
- `0 2px 8px rgba(0, 0, 0, 0.08)`
- 사용: 기본 카드, 입력 필드 (포커스)

#### LG (큰 그림자)
- `0 4px 12px rgba(0, 0, 0, 0.12)`
- 사용: 큰 카드, 호버 상태

#### XL (매우 큰 그림자)
- `0 8px 24px rgba(0, 0, 0, 0.15)`
- 사용: 모달, 드롭다운 메뉴

### 8.2 그림자 사용 가이드
- 과도한 그림자 사용 금지
- 깊이감 표현 시에만 사용
- 호버 시 그림자 증가로 인터랙션 표현

## 9. 애니메이션 및 트랜지션

### 9.1 트랜지션 속도

#### 빠름 (Fast)
- **시간**: 150ms
- **사용**: 호버 효과, 색상 변화

#### 보통 (Normal)
- **시간**: 200ms ~ 300ms
- **사용**: 기본 트랜지션, 페이드 인/아웃

#### 느림 (Slow)
- **시간**: 400ms ~ 500ms
- **사용**: 페이지 전환, 모달 열기/닫기

### 9.2 이징 함수
- **기본**: `ease-in-out`
- **부드러움**: `cubic-bezier(0.4, 0, 0.2, 1)`
- **바운스**: 사용하지 않음

### 9.3 애니메이션 가이드

#### 허용되는 애니메이션
- 페이드 인/아웃 (0.2초)
- 슬라이드 인/아웃 (0.3초)
- 색상 변화 (0.15초)
- 로딩 스피너 회전

#### 금지되는 애니메이션
- 과도한 바운스 효과
- 장식적인 애니메이션
- 자동 재생 비디오/이미지
- 깜빡이는 효과

## 10. 반응형 디자인

### 10.1 모바일 퍼스트 접근
- 모바일 디자인을 기본으로 설계
- 점진적 향상 (Progressive Enhancement)
- 터치 친화적 인터페이스

### 10.2 브레이크포인트 전략

#### 모바일 (320px ~ 767px)
- 단일 컬럼 레이아웃
- 세로 방향 네비게이션
- 큰 터치 타겟 (최소 44px × 44px)
- 간격 축소 (16px 기본)

#### 태블릿 (768px ~ 1023px)
- 2컬럼 레이아웃 가능
- 가로 방향 네비게이션
- 간격 증가 (20px 기본)

#### 데스크톱 (1024px 이상)
- 다중 컬럼 레이아웃
- 사이드바 옵션
- 간격 최대화 (24px 기본)
- 최대 너비 제한 (1200px)

### 10.3 터치 친화적 디자인
- 버튼 최소 크기: 44px × 44px
- 요소 간 최소 간격: 8px
- 스와이프 제스처 지원
- 더블 탭 방지

## 11. 접근성 (Accessibility)

### 11.1 색상 대비
- **WCAG AA 기준**: 최소 4.5:1 (일반 텍스트)
- **WCAG AAA 기준**: 최소 7:1 (작은 텍스트)
- 색상만으로 정보 전달 금지

### 11.2 키보드 네비게이션

- Tab 순서 논리적
- Esc 키로 모달/메뉴 닫기

### 11.3 스크린 리더 지원
- 의미 있는 HTML 시맨틱 태그 사용
- ARIA 레이블 적절히 사용
- 이미지에 alt 텍스트 제공
- 폼 요소에 라벨 연결

### 11.4 텍스트 크기
- 사용자 브라우저 설정 존중
- 최소 폰트 크기 16px
- 확대/축소 기능 지원
