# AI 신입생 도우미 서비스 기획안

## 1. 서비스 개요

### 1.1 서비스명
**AI 신입생 도우미**

### 1.2 대상 대학
**서울여자대학교**

### 1.3 서비스 목적
대학 신입생이 학사 일정, 공지사항, 지원 프로그램을 쉽게 이해하고 즉시 행동할 수 있도록 돕는 AI 챗봇 서비스입니다. 복잡한 행정·학사 용어를 쉬운 말로 풀어 설명하고, 신입생이 필요한 정보를 시기적절하게 제공하여 대학 생활 적응을 지원합니다.

### 1.4 핵심 가치
- **접근성**: 어려운 학사 용어를 쉬운 말로 변환하여 누구나 이해 가능
- **실행 가능성**: 정보 제공에 그치지 않고 구체적인 행동 가이드 제공
- **신뢰성**: 제공된 데이터만 기반으로 답변하여 정확한 정보 제공, 출처 표기로 신뢰도 향상
- **맞춤성**: 사용자의 전공, 학년, 시기 정보를 반영한 개인화된 안내
- **투명성**: 챗봇임을 명시하고 정보의 한계 및 출처를 명확히 제시

## 2. 사용자 페르소나

### 2.1 대상 사용자
**서울여자대학교 신입생**
- 입학 직후부터 1학년 1학기까지의 학생
- 대학 시스템과 학사 제도에 익숙하지 않은 상태
- 학사 일정, 행정 절차, 지원 프로그램에 대한 정보가 필요한 시기

### 2.2 사용자 특성
- **정보 니즈**: 학사 일정, 수강신청, 등록, 휴학 등 학사 관련 정보
- **행동 패턴**: 모바일 환경에서 빠르게 질문하고 즉시 답변을 받고 싶어함
- **심리적 특성**: 
  - 대학 생활에 대한 불안감과 기대감 공존
  - 실수를 두려워하여 정확한 정보를 원함
  - 복잡한 절차보다는 단계별 명확한 안내를 선호

### 2.3 사용 시나리오
1. **수강신청 전**: 수강신청 일정과 방법을 확인하고 싶을 때
2. **등록금 납부**: 등록금 납부 기간과 방법을 확인하고 싶을 때
3. **장학금 신청**: 자신이 신청 가능한 장학금 프로그램을 찾고 싶을 때
4. **공지 확인**: 중요한 학교 공지사항을 놓치지 않고 확인하고 싶을 때
5. **용어 이해**: 학사 관련 용어(예: 복수전공, 부전공, 휴학 등)를 이해하고 싶을 때
6. **프로그램 탐색**: 비교과 프로그램, 멘토링 프로그램 등 지원 가능한 프로그램을 찾고 싶을 때
7. **학점 제한 확인**: 수강신청 시 학점 제한(최소/최대)을 확인하고 싶을 때
8. **성적 우수자 학점**: 직전 학기 평점 평균 3.6 이상일 때 추가 신청 가능한 학점을 확인하고 싶을 때
9. **캠퍼스 마일리지**: 캠퍼스 마일리지 제도와 장학금 연계 방법을 알고 싶을 때
10. **학자금 대출**: 학자금 대출 신청 방법과 조건을 확인하고 싶을 때
11. **비교과 프로그램**: 학생성장지원시스템을 통한 비교과 프로그램 신청 방법을 알고 싶을 때
12. **멘토링 신청**: 온라인 동문 멘토링, 취업 멘토링 등 멘토링 프로그램 신청 방법을 확인하고 싶을 때
13. **SWU-SI 프로그램**: 사회혁신 프로젝트 참여 방법을 알고 싶을 때
14. **수강중도포기**: 수강신청 후 과목을 포기하고 싶을 때 절차를 확인하고 싶을 때
15. **오류 발생 시**: 챗봇이 답변하지 못할 때 인간 상담원 연결이 필요할 때

## 3. 핵심 기능 정의

### 3.1 학사 일정 안내 기능

#### 기능 설명
서울여자대학교의 학사 일정(수강신청, 등록, 휴학 신청, 성적 입력 등)을 조회하고 안내하는 기능입니다.

#### 주요 안내 항목
- 수강신청 일정 및 방법
  - 1학기: 개강 직전 및 개강 초기 정정 기간
  - 2학기: 8월 초 (예: 8월 3일~7일)
  - 수강신청 확인 및 정정 기간 (개강 직후)
  - 수강중도포기 기간 (개강 후 일정 기간)
- 등록금 납부 일정 및 방법
  - 학기별 등록금 납부 기간
  - 분할납부 안내
  - 미등록자 및 학자금대출자 최종납부 안내
- 휴학/복학 신청 일정 및 절차
- 성적 입력/확정 일정
- 학기별 주요 일정
  - 1학기 개강: 3월 초 (예: 3월 3일)
  - 2학기 개강: 9월 초 (예: 9월 1일)
  - 중간고사, 기말고사 일정
  - 방학 기간
- 계절학기 일정
  - 여름 계절학기 등록: 6월 초 (예: 6월 8일~12일)
  - 겨울 계절학기 일정
- 기타 학사 일정 (조기 졸업, 복수/부전공, 전과 신청 등)

#### 기능 특징
- 현재 시점에 해당하는 일정을 우선적으로 안내
- 과거/미래 일정은 "참고용"으로 구분하여 표시
- 일정의 중요도에 따라 우선순위를 부여하여 안내
- 학기별 주요 일정 (개강, 수강신청, 중간고사, 기말고사 등)을 명확히 구분하여 안내
- 계절학기 일정을 별도로 구분하여 안내

### 3.2 공지사항 안내 기능

#### 기능 설명
서울여자대학교의 주요 공지사항을 조회하고 안내하는 기능입니다.

#### 주요 안내 항목
- 학사 관련 공지
- 등록금 관련 공지
- 장학금 관련 공지
- 비교과 프로그램 공지
- 기타 중요 공지

#### 기능 특징
- 사용자가 관심 있어 할 만한 공지를 우선적으로 안내
- 공지의 중요도와 시급성을 판단하여 안내
- 공지 내용을 요약하여 핵심만 전달

### 3.3 지원 프로그램 안내 기능

#### 기능 설명
서울여자대학교에서 제공하는 각종 지원 프로그램(장학금, 비교과 프로그램, 멘토링 등)을 안내하는 기능입니다.

#### 주요 안내 항목
- 장학금 프로그램
  - 교외장학금 (외국교환학생 선발자 등, 우수한 학업성적 및 경제적 여건 고려)
  - 국가장학금 (국가보훈자녀, 북한이탈주민 자녀 등 특별 대상자)
  - 학교 내부 장학 (성적 우수 + 가정 형편 고려 등)
- 비교과 프로그램
  - 학생성장지원시스템 (SWU 비교과 통합관리 시스템)
    - 비교과 활동, 진로·취업, 상담 통합 관리
    - 핵심역량 진단 및 관리
    - 비교과 활동 기록 및 마일리지 시스템
  - SWU-SI (Social Innovation 교육센터)
    - 사회혁신자 양성을 위한 교과-비교과 연계 교육과정
    - 프로젝트, 봉사, 동아리 활동 포함
  - 창업동아리 (창업교육센터 지원)
- 멘토링 프로그램
  - 온라인 동문 멘토링
  - 취업 멘토링 (대학일자리플러스사업단)
  - 진로 컨설팅 및 실무 체험 기회
  - SWU-SI 프로젝트 멘토링
- 취업 지원 프로그램
  - 대학일자리플러스사업단 (취업경력개발팀, 현장실습지원팀)
  - 진로 및 취업 관련 정보 제공
- 캠퍼스 마일리지 제도 (서울여자대학교 특화 제도)
- 학자금 대출 제도 (한국장학재단 대출 등)
- 기타 지원 프로그램

#### 기능 특징
- 사용자의 전공, 학년, 시기 정보를 반영하여 맞춤 추천
- 신청 가능 여부와 신청 기간을 명확히 안내
- 신청 방법과 필요 서류를 구체적으로 안내
- 학생성장지원시스템을 통한 신청 절차 안내
- 장학금 종류별 자격 요건 및 혜택 상세 안내
- 비교과 프로그램별 특징 및 참여 방법 안내

### 3.4 신입생 기본 정보 안내 기능

#### 기능 설명
대학 생활에 필요한 기본적인 학사 구조와 용어를 설명하는 기능입니다.

#### 주요 안내 항목
- 학사 제도 설명 (학점제, 전공, 복수전공, 부전공 등)
- 학사 용어 설명 (휴학, 복학, 자퇴, 제적 등)
- 학사 구조 설명 (학기제, 성적 평가 방식 등)
- 행정 절차 설명 (각종 신청 방법 등)
- 서울여자대학교 특화 제도 설명
  - 학기별 수강 학점 제한
    - 최소 신청 학점: 10학점 (8학기 이내 재학생 필수)
    - 기본 최대 신청 학점: 18학점
    - 성적 우수자 (직전 학기 평점 평균 3.6 이상): 최대 21학점 신청 가능
    - Service Learning 및 사회봉사 과목: 최대 학점 + 1학점 추가 신청 가능 (성적 우수자 최대 22학점)
    - 학기 간 이월 학점: 1학기 잔여 학점 중 1~2학점을 같은 학년도 2학기로 이월 가능
  - 출결 규정 (수업일수의 3분의 1 이상 결석 시 학점 취득 불가)
  - 재수강 규정 (C+ 이하 성적 과목 재수강 가능)

#### 기능 특징
- 어려운 용어를 쉬운 말로 풀어서 설명
- 예시를 들어 이해하기 쉽게 설명
- 관련된 다른 정보와 연결하여 종합적으로 안내

## 4. 서비스 규칙 및 원칙

### 4.1 톤앤매너

#### 기본 톤
- **친절하고 차분함**: 신입생의 불안감을 이해하고 차분하게 안내
- **선배가 설명해주는 느낌**: 위에서 내려다보지 않고 옆에서 도와주는 느낌
- **공감과 이해**: 신입생의 입장에서 생각하고 답변

#### 언어 사용 원칙
- 어려운 행정·학사 용어는 반드시 쉬운 말로 풀어서 설명
- 예시를 들어 설명하여 이해도를 높임
- 불필요한 정보는 줄이고, 지금 필요한 정보 위주로 안내
- 과도하게 긴 답변은 지양하고 핵심만 전달

### 4.2 정보 사용 규칙

#### 데이터 기반 답변
- 반드시 제공된 학사 일정, 공지, 지원 프로그램 데이터만 기반으로 답변
- 데이터에 없는 내용은 추측하지 않음

#### 모르는 내용 처리
- 데이터에 없는 내용에 대한 질문 시 다음과 같이 답변:
  → "해당 내용은 현재 제공된 정보에서 확인되지 않아요. 학교 행정실이나 학과 사무실에 직접 문의해보시는 것을 권장드려요."
- 인간 상담원 연결 옵션 제공: AI가 처리할 수 없는 복잡한 질문이나 개인적인 상담이 필요한 경우, 인간 상담원과 연결할 수 있는 옵션 제공

#### 판단 및 조언 제한
- 학교·제도 관련 판단이나 개인적 조언은 하지 않음
- 객관적인 정보 제공에 집중

### 4.3 응답 구조 규칙

모든 답변은 아래 구조를 따릅니다:

#### 1. 한 줄 요약
- 지금 핵심만 간단히 요약
- 사용자가 즉시 파악할 수 있도록 명확하게 표현

#### 2. 자세한 설명
- 왜 중요한지 설명
- 무엇을 해야 하는지 구체적으로 안내
- 필요한 경우 관련 정보도 함께 제공

#### 3. 다음 행동 가이드
- 신청 방법 (어디서, 어떻게 신청하는지)
- 확인 사항 (준비할 서류, 확인할 사항)
- 준비할 것 (신청 전에 미리 준비해야 할 것)

#### 4. 출처 표기 (신뢰성 강화)
- 답변에 사용된 정보의 출처를 명시 (예: "2025학년도 학사일정", "학생처 공지사항" 등)
- 관련 링크나 참고 자료가 있는 경우 함께 제공
- 정보 업데이트 날짜 표기

### 4.4 질문 처리 방식

#### 질문 분류
질문을 아래 중 하나로 분류하여 처리:

1. **학사 일정** (수강신청, 휴학, 등록 등)
2. **공지 안내**
3. **지원 프로그램** (장학금, 비교과, 멘토링 등)
4. **신입생 기본 정보** (학사 구조, 용어 설명)

#### 모호한 질문 처리
- 질문이 모호하면 한 가지 추가 질문만 해서 명확히 함
- 예: "어떤 종류의 장학금에 대해 알고 싶으신가요? (신입생 장학금, 성적 장학금, 생활 장학금 등)"
- 선택지 버튼 제공: 사용자가 쉽게 선택할 수 있도록 버튼 형태의 선택지 제공
- 자동완성 제안: 질문 입력 중 자주 묻는 질문을 자동완성으로 제안

### 4.5 맞춤 추천 규칙

#### 사용자 정보 반영
- 사용자가 제공한 정보(전공, 학년, 시기)를 우선 반영
- 제공된 정보를 바탕으로 관련성 높은 정보를 우선적으로 안내
- 대화 이력 유지: 이전 대화 내용을 기억하여 문맥을 이해하고 연속적인 대화 가능
- 세션 내 정보 기억: 같은 세션 내에서 사용자가 제공한 정보(전공, 학년 등)를 기억하여 반복 질문 방지

#### 시기별 정보 구분
- 현재 시점에 해당하지 않는 정보는 "참고용"으로 구분하여 안내
- 현재 시점에 해당하는 정보를 우선적으로 안내

### 4.6 금지 사항

#### 추측성 답변 금지
- 데이터에 없는 내용을 추측하여 답변하지 않음
- 불확실한 정보는 제공하지 않음

#### 불확실한 일정·마감일 생성 금지
- 데이터에 없는 일정이나 마감일을 임의로 생성하지 않음
- 정확한 정보만 제공

#### 과도하게 긴 답변 금지
- 핵심 정보만 간결하게 전달
- 불필요한 설명은 생략

## 5. MVP 범위 정의

### 5.1 포함 기능

#### 필수 기능
1. **학사 일정 안내**: 수강신청, 등록, 휴학 등 주요 학사 일정 안내
2. **공지사항 안내**: 주요 공지사항 조회 및 안내
3. **지원 프로그램 안내**: 장학금, 비교과 프로그램, 멘토링 등 안내
4. **신입생 기본 정보 안내**: 학사 용어 및 제도 설명

#### 핵심 기능
- 자연어 질문 처리
- 질문 분류 및 적절한 답변 제공
- 맞춤 추천 기능 (전공, 학년, 시기 기반)
- 응답 구조화 (요약, 설명, 행동 가이드)
- 출처 표기 및 관련 링크 제공
- 대화 이력 유지 및 문맥 이해
- 선택지 버튼 및 자동완성 제안
- 오류 처리 및 인간 상담원 연결 옵션

### 5.2 제외 기능 (향후 확장)

#### 향후 추가 예정 기능
- 사용자 인증 및 개인화 프로필 관리
- 알림 기능 (중요 일정 알림, 공지 알림)
- 즐겨찾기 기능 (자주 찾는 정보 저장)
- 피드백 수집 기능 ("이 답변이 도움이 되었나요?" 등)
- 다국어 지원
- 음성 입력/출력 기능
- 접근성 기능 (화면 읽기, 키보드 조작, 색상 대비 등)
- 시스템 연동 (LMS, 학사관리시스템 등)
- 다중 채널 지원 (카카오톡, 라인 등 메신저 통합)

### 5.3 데이터 관리 방식

#### MVP 데이터 관리
- **수동 입력 방식**: 학사 일정, 공지사항, 지원 프로그램 정보를 수동으로 입력하여 관리
- 데이터 형식: 구조화된 데이터 형식으로 관리 (예: JSON, CSV 등)
- 데이터 업데이트: 정기적으로 데이터를 업데이트하여 최신 정보 유지

#### 데이터 항목
- 학사 일정 데이터
  - 일정명, 시작일, 종료일, 설명
  - 학기별 구분 (1학기, 2학기, 계절학기)
  - 일정 유형 (수강신청, 등록금 납부, 시험, 휴학/복학 등)
- 공지사항 데이터
  - 제목, 내용, 작성일, 중요도
  - 공지 유형 (학사, 등록금, 장학금, 비교과 프로그램 등)
  - 관련 부서 또는 학과
- 지원 프로그램 데이터
  - 프로그램명, 신청 기간, 대상, 신청 방법
  - 프로그램 유형 (장학금, 비교과, 멘토링, 취업 지원 등)
  - 자격 요건 및 필요 서류
  - 혜택 및 지원 내용
- 학사 용어 및 제도 설명 데이터
  - 용어 정의 및 쉬운 설명
  - 서울여자대학교 특화 제도 상세 설명
  - 관련 규정 및 절차 안내

## 6. 사용자 경험(UX) 설계 원칙

### 6.1 명확하고 간결한 대화 흐름
- 기능이나 목적을 초기에 명확히 표현
- 메시지는 짧고 이해하기 쉬운 언어 사용
- 지나치게 전문 용어 지양

### 6.2 예측 가능성과 선택지 제공
- 가능한 답변 버튼, 자동 완성, 추천 질문 제공
- 여러 경로로 같은 질문을 할 수 있게 허용 (예: "등록금 내는 방법" vs "수업료 결제 안내")

### 6.3 오류 처리 및 예외 대응
- 챗봇이 이해하지 못할 경우 정중하게 사과하고 대안 제시
- 사용자에게 질문을 다시 표현하도록 유도
- 인간 상담원 연결 옵션 제공

### 6.4 개인화 및 문맥 유지
- 사용자 정보(전공, 학년, 이전 대화 등)를 활용하여 맞춤형 답변 제공
- 대화의 문맥을 유지하여 반복 질문 방지 및 대화 연속성 보장

### 6.5 투명성과 윤리적 측면
- 챗봇임을 명시하고, 정보의 한계 및 출처를 알려줄 것
- 개인정보 보호 정책, 데이터 사용 방식에 대한 설명 제공
- 편견 없는 객관적인 정보 제공

### 6.6 일관된 목소리 및 브랜드 정체성 유지
- 서울여자대학교의 공식 어조를 유지하며 톤 일관성 확보
- 챗봇의 '성격' 또는 '페르소나'를 미리 정해두고 활용

## 부록: 주요 학사 용어 사전 (예시)

### A. 학사 제도 관련
- **학점제**: 과목을 수강하여 일정 학점을 취득해야 졸업할 수 있는 제도
- **전공**: 본인이 선택한 전문 분야
- **복수전공**: 두 개의 전공을 동시에 이수하는 것
- **부전공**: 전공 외에 추가로 선택하는 보조 전공
- **교양필수**: 모든 학생이 반드시 이수해야 하는 교양 과목
- **전공필수**: 전공 학생이 반드시 이수해야 하는 과목

### B. 학사 일정 관련
- **수강신청**: 원하는 과목을 선택하여 신청하는 것
- **등록**: 학기 등록금을 납부하고 학기를 시작하는 것
- **휴학**: 일정 기간 동안 학업을 중단하는 것
- **복학**: 휴학 후 다시 학교에 돌아오는 것
- **계절학기**: 방학 중에 운영되는 짧은 학기

### C. 성적 관련
- **학점**: 과목을 수강하여 받는 점수
- **평점평균(GPA)**: 모든 과목의 평균 점수
- **재수강**: 낮은 점수를 받은 과목을 다시 수강하는 것
- **이수**: 과목을 수강하고 학점을 취득하는 것

### D. 서울여자대학교 특화 용어
- **캠퍼스 마일리지**: 학생의 다양한 활동 및 학업 성취도를 마일리지로 적립하여 장학금 수혜 가능하게 하는 제도
- **학기별 수강 학점 제한**: 
  - 매 학기 최소 10학점 이상, 기본 최대 18학점 이하 신청 가능
  - 성적 우수자 (직전 학기 평점 평균 3.6 이상): 최대 21학점 신청 가능
  - Service Learning 및 사회봉사 과목: 최대 학점 + 1학점 추가 신청 가능
- **출결 규정**: 수업일수의 3분의 1 이상 결석 시 학점 취득 불가
- **학기 간 이월 학점**: 1학기 잔여 학점 중 1~2학점을 같은 학년도 2학기로 이월 가능
- **학생성장지원시스템**: 비교과 활동, 진로·취업, 상담을 통합 관리하는 시스템
- **SWU-SI**: Social Innovation 교육센터로, 사회혁신자를 양성하기 위한 교과-비교과 연계 교육과정

## 8. UI/UX 디자인 가이드

### 8.1 디자인 콘셉트

#### 전체 톤앤매너
- **신뢰감 있는 디자인**: 대학교 공식 사이트처럼 신뢰할 수 있는 느낌
- **학생 친화적**: 행정 사이트처럼 딱딱하지 않고 접근하기 쉬운 분위기
- **안정감**: "안내받고 있다"는 느낌이 드는 안정적인 구성
- **미니멀**: 과한 색상, 애니메이션, 장식 요소 사용 금지

#### 디자인 원칙
- **가독성 최우선**: 충분한 글자 크기 (모바일 최소 16px), 적절한 여백 (요소 간 최소 16px)
- **정보 과다 배치 금지**: 한 화면에 핵심 정보만 표시
- **명확한 흐름**: 질문 → 답변 흐름이 시각적으로 명확하게 보임
- **명확한 구분**: 버튼과 입력창이 명확하게 구분됨

### 8.2 컬러 & 스타일

#### 컬러 팔레트
- **메인 컬러**: 버건디 (Burgundy) `#800020`
  - 주요 버튼, 강조 요소에 사용
- **배경 컬러**: 화이트 `#FFFFFF`, 연한 그레이 `#F5F5F5` 또는 `#FAFAFA`
- **텍스트 컬러**: 주요 텍스트 `#1A1A1A`, 보조 텍스트 `#666666`

#### 타이포그래피
- **제목 (H1)**: 24px-28px, Bold, 행간 1.3
- **본문**: 16px, Regular, 행간 1.6
- **보조 텍스트**: 14px, Regular, 행간 1.5

#### 버튼 스타일
- **주요 버튼**: 버건디 배경, 화이트 텍스트, 12px 패딩, 8px 둥근 모서리
- **보조 버튼**: 화이트 배경, 버건디 테두리, 버건디 텍스트

### 8.3 레이아웃 구조

#### 전체 구조
```
┌─────────────────────────────────┐
│         Header (고정)            │
│  - 서비스명                      │
│  - 간단한 설명                   │
├─────────────────────────────────┤
│      Main Content Area          │
│  - 챗봇 대화 영역               │
│  - 질문 예시 버튼               │
├─────────────────────────────────┤
│      Input Area (고정)          │
│  - 질문 입력창                  │
│  - 전송 버튼                    │
├─────────────────────────────────┤
│      Footer / Side              │
│  - 자주 묻는 질문               │
│  - 학사 일정 보기               │
│  - 지원 프로그램 안내           │
└─────────────────────────────────┘
```

#### Header 영역
- **서비스명**: "AI 신입생 도우미" (좌측 상단)
- **설명 문구**: "궁금한 학사 정보를 쉽고 빠르게 물어보세요" (2줄 이내)
- 높이: 모바일 80px, 데스크톱 100px
- 배경: 화이트, 하단에 얇은 그림자 또는 구분선
- 고정 위치 (스크롤 시에도 상단 고정)

#### Main Content 영역
- **챗봇 대화 영역**: 화면 중앙, 모바일 100%, 데스크톱 최대 800px 중앙 정렬
- **메시지 스타일**:
  - 사용자 메시지: 우측 정렬, 버건디 배경, 화이트 텍스트
  - 봇 메시지: 좌측 정렬, 연한 그레이 배경, 다크 텍스트
- **질문 예시 버튼**: 대화 영역 상단 또는 입력창 위, 2~3개 제공
  - 예시: "수강신청은 언제 하나요?", "장학금 신청 방법이 궁금해요", "학사 용어가 어려워요"

#### Input Area (입력 영역)
- **위치**: 화면 하단 고정 (모바일)
- **높이**: 64px
- **구성**: 텍스트 입력 필드 + 전송 버튼
- **플레이스홀더**: "궁금한 것을 물어보세요 (예: 수강신청 일정, 장학금 신청 등)"

#### Footer / Side 영역
- **구성 요소**: 자주 묻는 질문, 학사 일정 보기, 지원 프로그램 안내
- **모바일**: Footer 형태
- **데스크톱**: Footer 또는 Sidebar 형태

### 8.4 반응형 디자인

#### Mobile First 접근
- **모바일 (320px ~ 768px)**: 세로 방향, 단일 컬럼, Header 고정, Input Area 하단 고정
- **태블릿 (768px ~ 1024px)**: 세로 방향, 여백 증가, Main Content 최대 700px 중앙 정렬
- **데스크톱 (1024px 이상)**: 중앙 정렬, 최대 너비 1200px, Main Content 최대 800px 중앙 정렬

#### 터치 친화적 디자인
- 버튼 최소 크기: 44px × 44px (터치 타겟)
- 요소 간 간격: 최소 8px

### 8.5 컴포넌트 설계

#### 주요 컴포넌트
- **Header**: 서비스명, 설명 문구
- **ChatArea**: 환영 메시지, 질문 예시, 메시지 리스트
- **InputArea**: 텍스트 입력창, 전송 버튼
- **Footer**: FAQ, 학사 일정, 지원 프로그램 링크

#### 메시지 컴포넌트
- **UserMessage**: 우측 정렬, 버건디 배경, 화이트 텍스트, 둥근 모서리 (12px)
- **BotMessage**: 좌측 정렬, 연한 그레이 배경, 다크 텍스트, 둥근 모서리 (12px), 출처 표기 (필요 시)

### 8.6 인터랙션 디자인

#### 사용자 플로우
**첫 방문자 플로우**:
1. 페이지 진입 → Header 확인 → 질문 예시 버튼 확인 → 질문 예시 클릭 또는 직접 입력 → 답변 확인 → 추가 질문 또는 Footer 링크 클릭

**재방문자 플로우**:
1. 페이지 진입 → 이전 대화 내역 확인 (선택) → 새로운 질문 입력 → 답변 확인

#### 상태 관리
- **로딩 상태**: 질문 전송 후 로딩 인디케이터 표시, 스켈레톤 UI 또는 스피너 사용
- **에러 상태**: 네트워크 오류 시 명확한 에러 메시지, 재시도 버튼 제공
- **빈 상태**: 첫 방문 시 환영 메시지, 질문 예시 제공

#### 애니메이션 (최소한)
- **허용**: 메시지 등장 페이드 인 (0.2초), 버튼 호버 색상 변화 (0.15초), 로딩 인디케이터 회전
- **금지**: 과도한 트랜지션, 장식적인 애니메이션
