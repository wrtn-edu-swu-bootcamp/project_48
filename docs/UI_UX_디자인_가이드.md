# AI 신입생 도우미 UI/UX 디자인 가이드

## 1. 프로젝트 개요

### 1.1 서비스명
**AI 신입생 도우미**

### 1.2 서비스 목적
신입생이 학사 일정, 공지, 지원 프로그램을 쉽고 빠르게 질문하고 안내받을 수 있는 챗봇 웹 서비스

### 1.3 타겟 사용자
- **대학 신입생** (서울여자대학교 신입생)
- 학사 용어와 학교 시스템에 익숙하지 않음
- 모바일 접속 비율이 높음
- 빠르고 정확한 정보를 원함

---

## 2. 디자인 콘셉트

### 2.1 전체 톤앤매너
- **신뢰감 있는 디자인**: 대학교 공식 사이트처럼 신뢰할 수 있는 느낌
- **학생 친화적**: 행정 사이트처럼 딱딱하지 않고 접근하기 쉬운 분위기
- **안정감**: "안내받고 있다"는 느낌이 드는 안정적인 구성
- **미니멀**: 과한 색상, 애니메이션, 장식 요소 사용 금지

### 2.2 디자인 원칙

#### 가독성 최우선
- 충분한 글자 크기 (모바일 최소 16px, 데스크톱 최소 14px)
- 적절한 여백 (요소 간 최소 16px 간격)
- 명확한 계층 구조

#### 정보 과다 배치 금지
- 한 화면에 핵심 정보만 표시
- 스크롤을 통한 자연스러운 정보 탐색
- 화면 분할을 통한 정보 구조화

#### 명확한 흐름
- 질문 → 답변 흐름이 시각적으로 명확하게 보임
- 대화형 인터페이스의 직관성 강조
- 사용자의 행동 경로가 명확함

#### 명확한 구분
- 버튼과 입력창이 명확하게 구분됨
- 클릭 가능한 요소와 텍스트의 시각적 차별화
- 상태 변화가 명확하게 표시됨

## 3. 컬러 & 스타일

### 3.1 컬러 팔레트

#### 메인 컬러
- **버건디 (Burgundy)**: `#800020` 또는 `#722F37`
  - 주요 버튼, 강조 요소에 사용
  - 서비스 브랜드 컬러

#### 배경 컬러
- **화이트**: `#FFFFFF`
  - 메인 배경
- **연한 그레이**: `#F5F5F5` 또는 `#FAFAFA`
  - 섹션 구분, 카드 배경

#### 텍스트 컬러
- **주요 텍스트**: `#1A1A1A` 또는 `#2C2C2C`
- **보조 텍스트**: `#666666` 또는 `#808080`
- **연한 텍스트**: `#999999`

#### 강조 색상
- **성공/확인**: `#28A745` (최소한으로 사용)
- **경고/주의**: `#FFC107` (최소한으로 사용)
- **에러**: `#DC3545` (최소한으로 사용)

### 3.2 스타일 가이드

#### 타이포그래피
- **제목 (H1)**: 24px-28px, Bold, 행간 1.3
- **제목 (H2)**: 20px-24px, Semi-Bold, 행간 1.4
- **본문**: 16px, Regular, 행간 1.6
- **보조 텍스트**: 14px, Regular, 행간 1.5
- **캡션**: 12px, Regular, 행간 1.4

#### 버튼 스타일
- **주요 버튼**: 버건디 배경, 화이트 텍스트, 12px 패딩, 8px 둥근 모서리
- **보조 버튼**: 화이트 배경, 버건디 테두리, 버건디 텍스트
- **텍스트 버튼**: 투명 배경, 버건디 텍스트, 언더라인 없음

#### 입력 필드 스타일
- **기본 입력창**: 화이트 배경, 연한 그레이 테두리, 12px 패딩, 8px 둥근 모서리
- **포커스 상태**: 버건디 테두리, 그림자 효과
- **플레이스홀더**: 연한 그레이 텍스트

```
├─────────────────────────────────┤

│  - 질문 입력창                  │

│      Footer / Side              │
│  - 자주 묻는 질문               │

```

### 4.2 Header 영역

#### 구성 요소
- **서비스명**: "AI 신입생 도우미" (좌측 상단)
- **설명 문구**: "궁금한 학사 정보를 쉽고 빠르게 물어보세요" (서비스명 하단, 2줄 이내)

#### 디자인 요구사항
- 높이: 모바일 80px, 데스크톱 100px
- 배경: 화이트
- 하단에 얇은 그림자 또는 구분선
- 고정 위치 (스크롤 시에도 상단 고정)

#### 예시 텍스트
```
AI 신입생 도우미
궁금한 학사 정보를 쉽고 빠르게 물어보세요
```

### 4.3 Main Content 영역

#### 챗봇 대화 영역
- **위치**: 화면 중앙
- **너비**: 모바일 100%, 데스크톱 최대 800px 중앙 정렬
- **패딩**: 좌우 16px, 상하 24px
- **스크롤**: 대화 내용이 많아지면 자동 스크롤

#### 메시지 스타일
- **사용자 메시지**: 우측 정렬, 버건디 배경, 화이트 텍스트, 둥근 모서리
- **봇 메시지**: 좌측 정렬, 연한 그레이 배경, 다크 텍스트, 둥근 모서리
- **메시지 간격**: 12px
- **메시지 최대 너비**: 80% (가독성 확보)

#### 첫 방문자용 질문 예시
- **위치**: 대화 영역 상단 또는 입력창 위
- **형태**: 버튼 형태, 2~3개 제공
- **스타일**: 보조 버튼 스타일 (화이트 배경, 버건디 테두리)
- **예시 질문**:
  - "수강신청은 언제 하나요?"
  - "장학금 신청 방법이 궁금해요"
  - "학사 용어가 어려워요"

### 4.4 Input Area (입력 영역)

#### 구성 요소
- **입력창**: 텍스트 입력 필드
- **전송 버튼**: 버건디 배경, 화이트 아이콘/텍스트

#### 디자인 요구사항
- **위치**: 화면 하단 고정 (모바일)
- **높이**: 64px
- **배경**: 화이트, 상단에 얇은 그림자
- **입력창**: 왼쪽 정렬, 충분한 패딩
- **전송 버튼**: 오른쪽 정렬, 원형 또는 둥근 사각형

#### 플레이스홀더 텍스트
"궁금한 것을 물어보세요 (예: 수강신청 일정, 장학금 신청 등)"

### 4.5 Footer / Side 영역

#### 구성 요소
- **자주 묻는 질문**: FAQ 링크
- **학사 일정 보기**: 학사 일정 페이지 링크
- **지원 프로그램 안내**: 지원 프로그램 페이지 링크

#### 디자인 요구사항
- **모바일**: Footer 형태, 하단 고정 또는 스크롤 가능
- **데스크톱**: Sidebar 형태 또는 Footer 형태
- **스타일**: 텍스트 링크, 버건디 색상, 호버 효과

#### 레이아웃 옵션
**옵션 1: Footer 형태**
```
[자주 묻는 질문] | [학사 일정 보기] | [지원 프로그램 안내]
```

**옵션 2: Sidebar 형태 (데스크톱)**
- 좌측 또는 우측에 세로 배치
- 아이콘 + 텍스트 형태

## 5. 반응형 디자인

### 5.1 Mobile First 접근

#### 모바일 (320px ~ 768px)
- **레이아웃**: 세로 방향, 단일 컬럼
- **Header**: 고정, 높이 80px
- **Main Content**: 전체 너비, 패딩 16px
- **Input Area**: 하단 고정, 높이 64px
- **Footer**: 하단 배치 또는 스크롤 가능

#### 태블릿 (768px ~ 1024px)
- **레이아웃**: 세로 방향, 여백 증가
- **Main Content**: 최대 너비 700px, 중앙 정렬
- **Input Area**: 하단 고정 또는 스크롤 가능

#### 데스크톱 (1024px 이상)
- **레이아웃**: 중앙 정렬, 최대 너비 1200px
- **Main Content**: 최대 너비 800px, 중앙 정렬
- **Sidebar**: 좌측 또는 우측에 추가 정보 배치 가능
- **Input Area**: 하단 고정 또는 인라인 배치

### 5.2 터치 친화적 디자인
- **버튼 최소 크기**: 44px × 44px (터치 타겟)
- **요소 간 간격**: 최소 8px
- **스와이프 제스처**: 대화 내역 스크롤 지원

## 6. 컴포넌트 설계

### 6.1 주요 컴포넌트

#### Header 컴포넌트
```jsx
<Header>
  <ServiceTitle>AI 신입생 도우미</ServiceTitle>
  <ServiceDescription>궁금한 학사 정보를 쉽고 빠르게 물어보세요</ServiceDescription>

```

#### ChatArea 컴포넌트
```jsx
<ChatArea>
  <WelcomeMessage />
  <QuestionExamples />
  <MessageList>
    <UserMessage />
    <BotMessage />
  </MessageList>
</ChatArea>
```

#### InputArea 컴포넌트
```jsx
<InputArea>
  <TextInput placeholder="궁금한 것을 물어보세요" />
  <SendButton />
</InputArea>
```

#### Footer 컴포넌트
```jsx
<Footer>
  <Link href="/faq">자주 묻는 질문</Link>
  <Link href="/schedule">학사 일정 보기</Link>
  <Link href="/programs">지원 프로그램 안내</Link>
</Footer>
```

### 6.2 메시지 컴포넌트

#### UserMessage
- 우측 정렬
- 버건디 배경 (`#800020`)
- 화이트 텍스트
- 둥근 모서리 (12px)
- 시간 표시 (선택)

#### BotMessage
- 좌측 정렬
- 연한 그레이 배경 (`#F5F5F5`)
- 다크 텍스트 (`#1A1A1A`)
- 둥근 모서리 (12px)
- 시간 표시 (선택)
- 출처 표기 (필요 시)

### 6.3 버튼 컴포넌트

#### PrimaryButton (주요 버튼)
- 버건디 배경
- 화이트 텍스트
- 호버 시 약간 어두워짐
- 클릭 시 피드백 애니메이션 (최소한)

#### SecondaryButton (보조 버튼)
- 화이트 배경
- 버건디 테두리
- 버건디 텍스트
- 호버 시 배경색 약간 변화

#### TextButton (텍스트 버튼)
- 투명 배경
- 버건디 텍스트
- 호버 시 언더라인

## 7. 인터랙션 디자인

### 7.1 사용자 플로우

#### 첫 방문자 플로우
```
1. 페이지 진입
   ↓
2. Header 확인 (서비스 이해)
   ↓
3. 질문 예시 버튼 확인
   ↓
4. 질문 예시 클릭 또는 직접 입력
   ↓
5. 답변 확인
   ↓
6. 추가 질문 또는 Footer 링크 클릭
```

#### 재방문자 플로우
```
1. 페이지 진입
   ↓
2. 이전 대화 내역 확인 (선택)
   ↓
3. 새로운 질문 입력
   ↓
4. 답변 확인
```

### 7.2 상태 관리

#### 로딩 상태
- 질문 전송 후 로딩 인디케이터 표시
- 봇이 답변을 생성하는 중임을 명확히 표시
- 스켈레톤 UI 또는 스피너 사용

#### 에러 상태
- 네트워크 오류 시 명확한 에러 메시지
- 재시도 버튼 제공
- 사용자 친화적인 에러 메시지

#### 빈 상태
- 첫 방문 시 환영 메시지
- 질문 예시 제공
- 사용 가이드 간단히 안내

### 7.3 애니메이션 (최소한)

#### 허용되는 애니메이션
- 메시지 등장: 페이드 인 (0.2초)
- 버튼 호버: 색상 변화 (0.15초)
- 로딩 인디케이터: 회전 애니메이션

#### 금지되는 애니메이션
- 과도한 트랜지션
- 장식적인 애니메이션
- 사용자 주의를 분산시키는 효과

## 8. 접근성 (Accessibility)

### 8.1 기본 접근성 요구사항
- **키보드 네비게이션**: 모든 인터랙티브 요소에 키보드 접근 가능
- **스크린 리더 지원**: ARIA 레이블 적절히 사용
- **색상 대비**: WCAG AA 기준 준수 (최소 4.5:1)
- **포커스 표시**: 키보드 포커스가 명확하게 보임

### 8.2 사용자 친화적 텍스트
- **간단한 언어**: 전문 용어 사용 최소화
- **명확한 라벨**: 버튼과 링크의 목적이 명확함
- **도움말 제공**: 필요 시 도움말 텍스트 추가
